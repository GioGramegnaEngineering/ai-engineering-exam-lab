# Repository Structure

This document defines the canonical structure for this portfolio repository.

## Root Layout

- `docs/`: methodology, policies, rubrics, and structural documentation.
- `prompts/`: reusable prompts for each stage of the AI-assisted workflow.
- `templates/`: reusable templates for case studies and technical reports.
- `case-studies/`: solved or in-progress engineering case studies.
- `assets/`: shared images and diagrams.

## Case Study Naming Convention

Case studies must be prefixed with a three-digit ID and kebab-case title.

Example:

- `001-edge-to-cloud-system-design`
- `002-smart-building-energy-management`

## Minimum Required Files for Each Case Study

- `README.md`
- `problem-statement.md`
- `assumptions.md`
- `solution.md`
- `verification.md`
- `limitations.md`
- `final-report.md`
