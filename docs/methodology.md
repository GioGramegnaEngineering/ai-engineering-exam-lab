# Methodology

This repository follows a structured engineering workflow for solving technical problems and exam-style case studies.

## Workflow

### 1. Problem Analysis

The problem statement is read and decomposed into:

- objective;
- input data;
- constraints;
- missing information;
- expected output;
- technical domain.

### 2. Assumptions

All missing or ambiguous elements are made explicit.

A good assumption must be:

- reasonable;
- technically defensible;
- clearly stated;
- compatible with the problem context.

### 3. Technical Decomposition

The problem is broken down into smaller subproblems.

Examples:

- system architecture;
- mathematical model;
- performance requirements;
- safety constraints;
- cost constraints;
- implementation constraints.

### 4. Solution Design

A solution is developed using engineering reasoning, calculations, diagrams, and documented decisions.

### 5. Verification

The solution is checked for:

- logical consistency;
- dimensional consistency;
- calculation errors;
- missing constraints;
- unrealistic assumptions;
- edge cases.

### 6. Critical Review

The solution is reviewed as if it were evaluated by an exam committee or a technical reviewer.

### 7. Final Report

The final output is rewritten as a clear technical report.
